1. INTRODUCTION                                             *fim_ollama_intro*

fim_ollama.vim provides inline code completions for Vim using Ollama or any
OpenAI-compatible text completions API. It renders ghost text as you type and
uses the same FIM prompt engineering as the FIM Completions VS Code extension.

Features:
  * Async completions using curl (no Python/Node required).
  * Vim 9 ghost text via |textprop|, with a popup fallback for older Vim.
  * File-level context: imports, definitions, recent code, scope hints.
  * Support for rnj-1, deepseek, qwen, gemma, mistral, and ministral FIM formats.

================================================================================
2. INSTALLATION                                             *fim_ollama_install*

With a package manager pointed at the `vim/` subdirectory (the plugin is not yet
extracted to its own repository):
>
    Plug 'guzmandrade-dev/ollama-fim.vim', { 'rtp': 'vim' }
<
Or with Vim packages:
>
    git clone https://github.com/guzmandrade-dev/ollama-fim.vim.git \
        ~/.vim/pack/plugins/start/ollama-fim.vim
<

3. CONFIGURATION                                             *fim_ollama_config*

Set these variables in your |.vimrc|.

*g:fim_ollama_api_url*
    API base URL.
    Default: "http://localhost:11434"

*g:fim_ollama_api_path*
    API endpoint path appended to the base URL.
    Default: "/api/generate"

*g:fim_ollama_backend*
    Request/response backend. Either "ollama" (default) or "openai" for
    OpenAI-compatible chat completions endpoints such as Together AI or
    local Ollama's /v1/chat/completions.

*g:fim_ollama_model*
    Model name passed to the API.
    Default: "rnj-1:8b-cloud"

*g:fim_ollama_model_type*
    FIM token family. One of: rnj-1, deepseek, qwen, gemma, mistral, ministral.
    Default: "rnj-1"

*g:fim_ollama_max_tokens*
    Maximum tokens to generate.
    Default: 64

*g:fim_ollama_temperature*
    Sampling temperature.
    Default: 0.1

*g:fim_ollama_include_file_context*
    Include file-level context (imports, definitions, comments).
    Default: 1

*g:fim_ollama_include_scope_info*
    Include "Currently in: ..." hint.
    Default: 1

*g:fim_ollama_file_context_chars*
    Max characters of file context.
    Default: 3000

*g:fim_ollama_api_key*
    API key sent as Bearer token. Falls back to $OLLAMA_API_KEY.
    Default: unset

*g:fim_ollama_normalize_indent*
    Normalize leading whitespace in completion text and context so it matches
    the current buffer's indent settings (expandtab, tabstop, shiftwidth).
    When 'expandtab' is set, tabs are expanded to spaces; when 'noexpandtab' is
    set, spaces aligning to shiftwidth are converted to tabs.
    Default: 1

================================================================================
4. COMMANDS                                                *fim_ollama_commands*

*:FimOllamaEnable*
    Turn completions on.

*:FimOllamaDisable*
    Turn completions off and cancel pending requests.

*:FimOllamaToggle*
    Toggle completions on/off.

================================================================================
5. MAPPINGS                                                *fim_ollama_mappings*

Default insert-mode mappings:
    <Tab>      Accept ghost suggestion.
    <M-]>      Cycle to the next suggestion (or request a new one from the model).

To customize, map before the plugin loads:
>
    imap <silent> <C-J> <Plug>(FimAccept)
    imap <silent> <C-L> <Plug>(FimNext)
<

================================================================================
6. MODEL FAMILIES                                            *fim_ollama_models*

The following families are supported, matching the prompt tokens each model
expects:

    rnj-1     <|pre_fim|> ... <|suf_fim|> ... <|mid_fim|>
    deepseek  <|fim_begin|> ... <|fim_hole|> ... <|fim_end|>
    qwen      <|fim_prefix|> ... <|fim_suffix|> ... <|fim_middle|>
    gemma     <|fim_prefix|> ... <|fim_middle|> ... <|fim_suffix|>
    mistral   <s>[SUFFIX]{suffix}[PREFIX]{prefix} (raw mode)
    ministral <s>[SUFFIX]{suffix}[PREFIX]{prefix} (chat-template-wrapped)

For `mistral` the request is sent with Ollama `raw` mode so the model's
chat template is not applied. For `ministral`, Ollama Cloud currently
rejects `raw` mode, so the plugin embeds the SPM markers inside a short
code-completion instruction. Use |g:fim_ollama_system_prompt| to customize
that instruction.

When using the "openai" backend, set |g:fim_ollama_api_path| to the chat
completions endpoint for your provider (e.g. "/v1/chat/completions" on
Together AI or local Ollama). The FIM prompt is sent as the user message
and the generated code is read from `choices[0].message.content`. The
FIM prompt tokens are still controlled by |g:fim_ollama_model_type|.
Use |g:fim_ollama_system_prompt| to customize the system message for the
chat backend.

*g:fim_ollama_system_prompt*
    System/instruction prompt used for chat-template-wrapped models such
    as `ministral`, and for the "openai" chat-completions backend.
    Default: see autoload/fim_ollama/prompt.vim
